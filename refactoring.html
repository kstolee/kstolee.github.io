Refactoring Pipe-like Mashups: Preliminary Study

In a preliminary study of the impact of refactoring on pipe-like mashups, we evaluated a smaller set of 200 Pipe-like mashups and manually inspected the end-to-end transofmrations of those pipes. We make this experimental infrastructure available here, and detail our experience in a Technical Report.

Experimental Infrastructure

Download the executable for the experimental infrastructure: Experimental.zip The infrastructure runs with the following java configuration:

     java version "1.6.0_15"
     Java(TM) SE Runtime Environment (build 1.6.0_15-b03)

Unzip the archive, and there will be two directories and a runnable jar. The directory 'orig/' contains the original pipes used in this experiment. The directory 'trans/' is initially empty. It will be populated once the experimental infrastructure is executed.

To run all* refactorings on all pipes: 

                            java -jar Pipes.jar

The command-line output will have seven columns: the pipe name, the number of modules in the original pipe, the number of modules in the transformed pipe, the number of parameters in the original pipe, the number of parameters in the transformed pipe, the ratio of wired parameters to modules in the original pipe, and the ratio of wired parameters to modules in the transformed pipe.

The 'trans/' directory will now contain the refactored pipes.

To run all* refactorings on one pipe: 

                            java -jar Pipes.jar -f <infile> <outfile>

The output will have the same seven columns described above. Additionally, the <outfile> will contain the JSON representation of the transformed pipe.

Experimental Artifacts

Each pipe is represented in JSON and stored in a text file, XXX.txt where 'XXX' is the pipe number. To make the pipe easier to read, we also encode the pipe structure in a graph using the DOT language. Each Pipe has a corresponding .dot file.
