Research Interests

My research interests are in software engineering, focusing on three primary areas: 1) code search, 2) program repair, 3) regular expression testing and analysis, 4) testing education, and 5) empirical studies. Although code search is an integrated part of modern development practices, current search tools support textual queries but ignore a critical component of source code: its behavior. During my PhD, I developed a code search approach based on behavior that uses symbolic execution and a constraint solver to identify exact or close matches to an input/output query. While effective at finding relevant code snippets for reuse, semantic code search is also an effective engine behind program repair. The patches located through semantic code search have measurably higher quality compared to prior techniques. One source of bugs that is difficult to repair comes from regular expressions, and these bugs can be catastrophic. Repairability is limited in part because regular expressions are under-tested; we am developing techniques to improve regular expression quality.
